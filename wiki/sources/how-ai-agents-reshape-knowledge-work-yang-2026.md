# How AI Agents Reshape Knowledge Work: Autonomy, Efficiency, and Scope

Surfaced via the NLP newsletter (Top AI Papers of the Week, 2026-06-07–06-14).

## Summary

An economics paper using large-scale production data from Perplexity to study the transition from conversational assistants to autonomous agents. It compares **Search** (a conversational assistant) against **Computer** (a general-purpose agent) along three axes — autonomy, efficiency, and task scope. The authors frame the shift with a task-based cost model in which agents carry a higher fixed *delegation cost* but a lower marginal *execution cost per step*; this predicts that agent access expands the affordable task frontier toward higher-value work and (weakly) raises realized value and surplus. The empirical findings, drawn from matched session pairs and matched tasks as natural experiments, broadly confirm the model: agents accelerate workflows, raise output quality, cut costs, and broaden the scope of work users attempt.

## Key points

- **Autonomy:** Computer performs ~26 minutes of autonomous work per session versus ~33 seconds for Search, automating task decomposition and execution that Search users would orchestrate manually.
- **Quality:** Per-query medium-to-high dissatisfaction rates are 55% lower on Computer than on Search; the follow-up query mix shifts toward higher-order work (verification, extension).
- **Efficiency:** On matched tasks, Computer cuts completion time from 269 to 36 minutes — an estimated 87% time reduction and 94% cost reduction versus a human equipped with Search alone.
- **Scope:** Computer queries more often cross occupational boundaries, demand higher-order cognition, draw on broader expertise, and bundle multiple subtasks into single composite queries; some work activities are essentially absent from Search usage.
- **Method:** Identification leans on matched session pairs with near-identical initial queries and matched tasks across the two products to approximate the same underlying task attempted both ways.
- A cost-structure framework (fixed delegation cost vs. marginal execution cost) is the analytical spine connecting the model to the data.
